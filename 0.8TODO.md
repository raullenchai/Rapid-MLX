  - Repro: `tools=[{"type":"function","name":"f","parameters":"not object"}]`
  - Actual: Chat tool function receives `parameters: "not object"`.
  - Expected: reject strings/lists/scalars; `parameters` must be an object JSON Schema.
  - Impact: malformed tool schemas can reach prompt/template code.

- Responses reasoning: `reasoning.effort` is accepted but ignored, including invalid values.
  - Repro: `reasoning={"effort":"low"}`, `{"effort":"banana"}`, or `{"effort":123}` through `responses_to_openai`.
  - Actual: all convert with `reasoning_max_tokens=None`, `enable_thinking=None`, no template kwargs.
  - Expected: validate supported values and map them to reasoning controls, or explicitly reject unsupported controls.
  - Impact: Responses clients get no signal that requested reasoning controls do nothing.

### Engine / Cache / MLLM Perf

- Prefix-cache perf: `MemoryAwarePrefixCache.fetch()` degenerates on long shared-prefix misses.
  - Repro: populate 1000 `_sorted_keys` of length 8192 sharing the first 8191 tokens, then fetch a divergent key.
  - Actual: about 15 ms per miss on local Python 3.12; cost scales with entries x prefix length.
  - Expected: bounded near `O(log N + small candidate window)`, or index prefix lengths/trie nodes instead of scanning same-first-token ranges.
  - Impact: agent/tool workloads with many long shared prompts pay avoidable admission/TTFT CPU overhead.

- MLLM perf: stop-string enforcement re-decodes cumulative output every token.
  - Status: FIXED locally in `00b5982` (`Reduce streaming parser hot-path overhead`).
  - Repro: drive `MLLMScheduler._process_batch_responses()` with 16k fake tokens and `stop=["NEVER_MATCH"]`.
  - Actual: local microbench 0.019s without stop vs 3.91s with stop, about 201x slower.
  - Expected: reuse streaming decoded buffer or rolling suffix matcher; inspect only the new tail needed for stop matching.
  - Impact: VLM/MLLM streaming requests with stop sequences can burn CPU and inflate per-token latency on long outputs.

- Engine route contract: `tests/test_route_engine_contract.py::test_no_hasattr_engine_guard` fails for `routes/completions.py`.
  - Repro: `/opt/homebrew/bin/python3.12 -m pytest tests/test_route_engine_contract.py::test_no_hasattr_engine_guard -q`
  - Actual: `hasattr(engine, "stream_generate")` at completions route line 185 violates the route contract guard.
  - Expected: hoist `stream_generate`/logprobs support onto `BaseEngine` or use explicit capability flags.
  - Impact: engine API drift can silently change completions behavior instead of failing through a stable contract.
